# Glossary

- **Tenant** – A single school; identified by a unique `schoolId`.
- **Super Admin** – Platform-wide admin with cross-tenant read/write access.
- **RBAC** – Role-Based Access Control; defines permissions per role.
- **Clean Architecture** – Separation of concerns ensuring business logic is independent of frameworks.
- **Next.js** – React framework for server-rendered and static web applications.
- **Prisma** – Type-safe ORM for PostgreSQL.
- **Server Component** – React component rendered on the server (Next.js).
- **Rate Limiting** – Restricts number of API calls per user per minute.
- **WCAG 2.1 AA** – Web Content Accessibility Guidelines, level AA.
- **API Versioning** – Prefixing endpoints with `/api/v1/` to manage breaking changes.
- **NeonDB** – Serverless PostgreSQL platform.
- **shadcn/ui** – Reusable React component library built on Tailwind CSS.
- **Vercel** – Cloud platform for frontend frameworks and static sites.
- **SaaS** – Software-as-a-Service; multi-tenant subscription delivery model.
